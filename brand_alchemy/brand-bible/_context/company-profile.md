# Company Profile

_Last updated: 2026-05-13_

## Company

rkitect.ai is an AI-powered architectural rendering and visualization platform built by HappyMonk AI in Bengaluru, India.

## Current Narrative

The platform is positioned as "Designed for real architects." It focuses on fast visual generation and controlled revisions, not generic AI image creation.

## Product Promise

Transform sketches, photos, floor plans, 3D models, or text prompts into photorealistic, client-ready architectural visuals quickly, then revise individual elements without restarting the workflow.

## Current Strategic Wedge

Post-render AI segmentation is the strongest current differentiator. It allows users to click rendered elements such as walls, floors, sofas, or lights and modify them without a full re-render.

## Working Product Brief

- AI rendering from sketches, photos, floor plans, 3D models, or prompts.
- Multi-style generation across architecture and interior styles.
- Floor plan to 3D render workflow.
- Video walkthrough generation.
- Presentation board export.
- Workflow designed for teams that do not want to rely on full 3D software for every concept iteration.

## Unique Competitive Moat: Material & Object Segmentation

### The Feature: How It Works (6-Step Workflow)

1. User uploads input (sketch, photo, floor plan, CAD, or prompt).
2. rkitect.ai generates photorealistic render in seconds.
3. AI immediately reads and analyzes the rendered output.
4. AI automatically identifies and labels every major object: walls, windows, doors, flooring, furniture, lighting, materials.
5. User clicks any labeled object and modifies it: change material, color, style, or swap item entirely.
6. Changes apply instantly without re-rendering the entire scene.

### Why This Is Unique

Every competitor (Veras, LookX, mnml.ai, MyArchitectAI, ArchiVinci, Coohom) requires users to manually identify areas they wish to change:

- Manual masking or painting around objects (45–90 seconds per object).
- User must write new prompt or select new style.
- System re-renders the entire image.
- Result: Often hallucinated or structurally altered.

**rkitect.ai inverts this:** AI identifies first. User clicks second. The render stays structurally intact.

### Competitive Moat Analysis

**What competitors would need to build to match this:**

- Post-render image segmentation pipeline (3–6 months).
- Real-time object labeling and semantic understanding (4–8 months).
- Non-destructive editing engine that respects spatial relationships (4–6 months).
- Full integration with render engine for instant material/color swaps (2–3 months).
- Testing and reliability across diverse architectural inputs (2–4 months).

**Time-to-copy estimates:**

- ArchiVinci, mnml.ai, MyArchitectAI: 12–18 months (have rendering pipeline, need segmentation).
- Veras, LookX: 9–15 months (have 3D model integration advantage).
- New competitors: 18–24 months (must build from scratch).

### Recommended Feature Name: Sektura

**Reasoning:**

- Phonetic: Strong plosive/authority (K, T sounds).
- Semantic: Suggests "section," "sector," "segmentation" without using jargon.
- Memorable: One word, easy to spell, available domain.
- Positioning: "Sektura reads your render and segments every element for instant editing."

**Register and brand:** sektura.ai, sektura.com are confirmed available. Every UI element involving post-render segmentation should carry the Sektura mark.
